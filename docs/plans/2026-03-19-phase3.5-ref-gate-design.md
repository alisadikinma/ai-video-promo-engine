# Design: Phase 3.5 — Reference Image Validation Gate + Multi-Character Cast System

**Date:** 2026-03-19
**Status:** Approved

---

## Problem Statement

Current pipeline allows proceeding to Phase 4 (Image Prompts) and Phase 5 (Video Prompts) without mandatory reference images for product, environment, characters, and costumes. This leads to inconsistent AI-generated visuals. Additionally, the system assumes a single creator/presenter, but real promotional videos often need multiple characters with different roles.

---

## Design Decisions (from Brainstorm)

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Gate position | New Phase 3.5 (between Scene Breakdown and Image Prompts) | At this point we know exactly which scenes exist, so we can auto-derive required refs |
| Ref categories | All 5 mandatory (Character, Product, Environment, Brand, Costume) | Hard block on all |
| Detection method | Hybrid (auto-derive from scene plan + user confirm) | Best balance of automation and user control |
| Block policy | Hard block — no skip, no override | Zero tolerance for missing refs |
| Cast system | Max 5 characters (1-3 utama + 0-2 pendamping) | Matches NB2 identity lock limit |
| Cast ref treatment | Utama = full ref (face+body+costume), Pendamping = minimal (face only) | Proportional to screen time and identity tracking needs |
| Costume detection | Auto-detect institution from brand/product + confirm with user | Handles KAI, Pelindo, BRI, etc. |
| Costume ref count | 1 foto minimum (front view) | Lightest user burden while sufficient for AI |

## Architecture

### Pipeline Update

```
Phase 3: SCENE BREAKDOWN       → Output: scene-plan.md
  └─ [USER APPROVAL GATE]
                ↓
Phase 3.5: REFERENCE COLLECTION → Output: ref-manifest.md  ← NEW
  ├─ Auto-scan scene-plan.md
  ├─ Derive required ref images per scene
  ├─ Present checklist to user via AskUserQuestion
  ├─ User uploads to {project}/ref/
  ├─ Engine validates ALL refs exist
  └─ [HARD BLOCK — cannot proceed until 100% complete]
                ↓
Phase 4: IMAGE PROMPTS (NB2)    → Output: image-prompts.md
```

### Multi-Character Cast System (Replaces Single-Creator)

```
CAST (max 5 characters total)
├── Pemeran Utama (1-3 orang)
│   ├── WAJIB: face ref (front)
│   ├── WAJIB: full body ref
│   ├── WAJIB: costume/uniform ref (if institutional)
│   └── Identity lock: FULL (tracked across ALL scenes)
│
└── Pemeran Pendamping (0-2 orang)
    ├── WAJIB: face ref (front)
    ├── OPTIONAL: body/costume (derive dari deskripsi)
    └── Identity lock: PARTIAL (tracked in assigned scenes only)
```

### Institution-Aware Costume System

```
IF brand/product linked to specific institution:
    → Auto-detect: "KAI" → railway uniform
    → Auto-detect: "Pelindo" → port authority uniform
    → Auto-detect: "BRI" → banking uniform
    → Auto-detect: "RS/Hospital" → medical uniform
    → Auto-detect: "Pertamina" → energy sector uniform

    → AskUserQuestion: "Terdeteksi produk ini untuk {institution}.
       Apakah karakter di video perlu pakai seragam {institution}?"

    → IF Ya → flag: costume_type = "institutional"
    → ALL cast members in scenes → WAJIB costume ref
```

## Naming Convention

```
ref/cast-c{N}-face.png        → Character N face (front view)
ref/cast-c{N}-body.png        → Character N full body
ref/cast-c{N}-costume.png     → Character N costume/uniform
ref/product-{name}.png        → Product reference
ref/env-{location}.png        → Environment reference
ref/brand-{asset}.png         → Brand asset
ref/costume-{institution}.png → Institution uniform (shared reference)
```

---

## Auto-Derive Logic (Phase 3.5)

| Scene Element | Detected Via | Required Ref Category |
|---|---|---|
| Character appears | Scene Type = Presenter, Dialogue = Yes | Cast ref (per character) |
| Product shown | Script mentions product name/feature | Product ref |
| Specific location | Scene Type = B-Roll, environment description | Environment ref (per unique location) |
| Brand visible | Script mentions brand, demo, dashboard | Brand asset ref |
| Institutional context | Brand/product → institution mapping | Costume ref |

## NB2 Prompt Impact (Multi-Character)

Multi-character scenes now specify each character:

```
SUBJECT: Character 1 ({name}) and Character 3 ({name}) in conversation.
Using facial identity from ref/cast-c1-face.png for Character 1.
Using facial identity from ref/cast-c3-face.png for Character 3.
Character 1: {description}... wearing {costume from ref}.
Character 3: {description}... wearing {costume from ref}.
INTERACTION: {scene action between characters}.
```
